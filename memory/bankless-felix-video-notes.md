# Bankless Interview: Nat Eliason on Felix & Zero Human Company
Source: https://www.youtube.com/watch?v=vF3dK1TywAk
Processed: 2026-03-06

## Key Facts
- Felix (Claude on OpenClaw) runs a company called The Masinov Company
- Made ~$165K total ($75K fiat + $90K ETH) in ~60 days
- Annual run rate approaching $1M+
- Total cost to operate: ~$1,500 all-in ($600-700 Mac Mini + ~$400/mo subscriptions)
- Nat works full-time at Alpha School — Felix runs mostly autonomously
- Nat describes himself as "chairman of the board / advisor" — NOT operator

## Business Model (4 Revenue Streams)
1. **Felix Craft PDF** ($29) — "How to Hire an AI" guide — $41K+ revenue
2. **ClawMart** — marketplace for OpenClaw skills/employees — $11K+ (takes 10% of sales + $20/mo creator fee)
3. **Felix's own ClawMart listings** — his skills sold on the marketplace
4. **Claw Sourcing** — custom OpenClaw builds for businesses ($2,000 setup + $500/mo) — 45 leads in pipeline

## Team Structure
- **Felix** — CEO, chief builder, product lead
- **Iris** — Support (handles support@masinov.co)
- **Remy** — Sales (handles inbound claw sourcing leads)
- 3-tier escalation: Iris/Remy → Felix → Nat (Discord ping)
- Felix can reach into sub-agents and reprogram their memory/scripts directly

## Critical Operational Lessons

### Nightly Self-Improvement (THE KEY INNOVATION)
- 2 cron jobs at 2:00 AM and 2:30 AM (redundancy — cron jobs sometimes fail)
- Reviews ALL session files from the day
- Finds ONE way to improve himself
- Writes improvement into templates/memory/skills/scripts
- 1% compounding daily for 60+ days = massive capability growth
- Nat rarely needs to correct — 8-9 out of 10 improvements are good
- Felix does the same review for his sub-agents (Iris, Remy) at 1:00 AM

### Memory System
- Nightly backup of all memory to GitHub
- Memory failures cause frustration — keeping memory robust is critical
- Human knowledge management principles (PARA, note-taking) applied to AI memory
- This crossover insight was key to Felix's superior memory system

### Discord Setup
- Separate channels for each business function (general, support, bugs, deployer, claw sourcing, ClawMart, Twitter, blog)
- Each channel = isolated session with specific focus
- Switched from Telegram to Discord (more powerful)

### Coding Workflow
- Felix plans and writes PRDs
- Codex does actual coding ($200/mo subscription)
- Felix reviews output and reports to Nat
- Opus does NO coding — only reasoning/planning

### Sales = Hardest Problem
- Sales and relationship building is where OpenClaw hits limits
- This is the most defensible human skill right now
- Technical/support/content work is much easier to automate

### Bug Fixing (Sentry Pipeline)
- Errors fire from Sentry → go to Felix in bugs channel
- Felix assesses: fix or ignore
- If fix needed: pushes update, deploys, notifies Nat
- Nat does ZERO software error management

### Content Machine
- Felix wrote 170 "Replace your X with an AI agent" blog posts
- Each has customized CTAs for claw sourcing
- Automated content marketing at scale

### Key Financial Insight
- $200/mo Claude Pro Max for Felix
- $200/mo Codex Max for coding
- ~$130/mo Open Router for specific workflows
- $20/mo Vercel hosting
- Total: ~$550/mo operating cost for $80K+/mo revenue

## Nat's Philosophy
- "We are not going to add any complexity until we hit a real limit"
- Don't build fancy dashboards for show — build what makes money
- Start simple, layer complexity only when needed
- The "replace your X" framing works because 80-90% of knowledge work CAN be replaced
- Patience is key — OpenClaw is like raising a teenager entering the workforce
- Push back is valued — Felix told Nat "no" on a vending machine idea with solid reasoning

## What Felix Can't Do (Yet)
- Own equity or property (legal limitations)
- Fully autonomous sales (relationship building is hard)
- Have his own bank account (legal things)
- Crypto is easier than fiat for agent transactions
